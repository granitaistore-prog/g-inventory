# Реєстр граніту — облік складу

Веб-програма для обліку плитки, заготовок і пам'ятників. Дані зберігаються
у файлі `data.json` прямо у вашому GitHub-репозиторії — окремої платної
бази даних не потрібно. Програма відкривається в браузері (комп'ютер,
телефон), а пізніше її можна буде «загорнути» у застосунок для Android/iPhone
без переписування коду (див. розділ нижче).

## Що вміє

- Таблиця складу: тип, матеріал, колір, розміри, кількість, ціна, локація,
  дата надходження, примітки.
- Додавання / редагування / видалення позицій.
- Пошук і фільтри за типом та локацією, сортування по колонках.
- Позначення позицій, де залишок нижче мінімального (червоним).
- Синхронізація з GitHub: кожна зміна одразу зберігається в репозиторії
  як окремий коміт (видно історію — хто, що і коли змінив).
- Працює з кількох місць одночасно (офіс + виїзд) — див. нижче, як це
  обробляється.
- Можна встановити на телефон як застосунок (іконка на головному екрані).

## Важливо розуміти про зберігання даних

Ви обрали найпростіший і безкоштовний варіант — файл у GitHub. Це чудово
підходить для невеликої команди, але має одну особливість: якщо двоє людей
одночасно зберігають зміни, GitHub приймає лише першу і відхиляє другу
(захист від випадкового затирання чужої роботи). Програма сама це ловить
і автоматично: підвантажує свіжі дані, повторно застосовує саме вашу зміну
і зберігає ще раз. Це надійно працює, коли двоє людей редагують різні
позиції складу. Якщо ж двоє одночасно (в межах секунд) редагують одну й ту
саму позицію — переможе той, хто зберіг останнім, і про це варто пам'ятати.
Для невеликого бізнесу з обліком складу це, як правило, не проблема.

Якщо в майбутньому команда виросте і одночасних правок стане багато,
є сенс перейти на легку хмарну базу (Firebase/Supabase) — структура
програми це дозволяє зробити без переробки інтерфейсу.

## Крок 1. Створити репозиторій на GitHub

1. Зайдіть на github.com → New repository.
2. Назвіть його, наприклад, `granite-inventory`. Публічний чи приватний —
   на ваш вибір (приватний надійніше, якщо не хочете, щоб хтось сторонній
   бачив ваш склад і ціни).
3. Завантажте в цей репозиторій усі файли з цієї теки: `index.html`,
   `style.css`, `app.js`, `manifest.json`, `icon.svg`, `data.json`.

## Крок 2. Увімкнути GitHub Pages (щоб отримати посилання на сайт)

1. У репозиторії: Settings → Pages.
2. Source: "Deploy from a branch", Branch: `main`, папка `/ (root)`.
3. Збережіть. Через хвилину-дві GitHub видасть посилання на кшталт
   `https://ваш-логін.github.io/granite-inventory/`.
4. Це посилання відкривайте на будь-якому пристрої — комп'ютер, телефон
   офісу чи виїзної бригади.

## Крок 3. Створити токен доступу (Personal Access Token)

Токен потрібен, щоб програма могла записувати зміни у ваш репозиторій.

1. GitHub → фото профілю → Settings → Developer settings →
   Personal access tokens → Fine-grained tokens → Generate new token.
2. Repository access: оберіть лише ваш репозиторій `granite-inventory`
   (не всі репозиторії підряд — це безпечніше).
3. Permissions → Repository permissions → Contents → **Read and write**.
4. Термін дії — оберіть зручний (можна щорічно оновлювати).
5. Створіть токен і скопіюйте його (він показується один раз).

Кожна людина, яка користується програмою з іншого пристрою/браузера,
створює **свій власний токен** і вводить його у своїх «Налаштуваннях» —
токен зберігається тільки в браузері цього пристрою.

## Крок 4. Підключити програму

1. Відкрийте посилання з Кроку 2.
2. Натисніть «⚙ Налаштування».
3. Заповніть: власник репозиторію (ваш логін GitHub), назва репозиторію,
   гілка (`main`), шлях (`data.json`), токен.
4. «Зберегти й підключитись» — таблиця завантажиться (спочатку порожня).

Готово — можна додавати позиції складу.

## Встановлення на телефон (вже зараз)

- **Android (Chrome):** відкрийте посилання → меню (три крапки) →
  «Додати на головний екран».
- **iPhone (Safari):** відкрийте посилання → кнопка «Поділитися» →
  «На екран «Домівка»».

Іконка з'явиться як окремий застосунок, що відкривається на весь екран.

## План на майбутнє: повноцінні застосунки для App Store / Google Play

Коли буде потрібен «справжній» застосунок у сторах (наприклад, для пуш-сповіщень
про закінчення матеріалу), цей самий код можна загорнути інструментом
**Capacitor** — він бере наявний сайт і збирає з нього нативний Android/iOS
проєкт без переписування логіки складу. Це наступний крок, а не заміна:
вебверсія і мобільна працюватимуть з тими самими даними в GitHub.

## Структура файлів

```
index.html      — розмітка сторінки
style.css       — оформлення
app.js          — логіка: робота з GitHub API, таблиця, форми
manifest.json   — налаштування для встановлення на телефон
icon.svg        — іконка застосунку
data.json       — початковий порожній файл даних (сюди пишуться позиції складу)
```
